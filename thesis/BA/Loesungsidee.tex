\chapter{Entwicklung einer Lösungsidee}\label{Entwicklung einer Lösungsidee}
\section{Allgemeine Beschreibung der Lösungsidee}\label{AllgBeschreibLösung}
Eine Lösungsidee für die in Kapitel \ref{Probleme}
beschriebenen Probleme wurde im Kapitel \ref{Ziel} bereits angedeutet. Kern dieser Idee
ist, die im vorherigen Kapitel angesprochene \g zur Beschreibung von
\guis zu nutzen.
Diese \guis sollen so beschrieben werden, dass sie in der Domäne von \pcs
für unterschiedliche \emph{GUI-Frameworks} genutzt werden können. Diese
Beschreibung soll weiterhin nur einmal stattfinden. Der Quellcode, welcher die \gui im entsprechenden
Framework darstellt, wird frameworkspezifisch aus der
\emph{GUI-Beschreibung} generiert.
Langfristig betrachtet könnte das \MCF damit abgelöst werden.\\
Die Anforderungen für die \g wurden bereits beschrieben. Diese
sollen so weit wie möglich im Prototypen, welcher im Zuge dieser Arbeit entwickelt wird,
umgesetzt werden. Der Prototyp soll
Quellcode erzeugen, der eine \gui mit den syntaktischen Strukturen des \MCF
beschreibt. Somit kann geprüft werden, ob sich der generierte Quellcode in
\pcs einbinden lässt.
\section{Konzept}
Die \g  wird für die abstrakte
Beschreibung der \gui verwendet. Somit ist gewährleistet, dass die \gui weiterhin
nur einmal beschrieben werden muss. 
Wie in Abschnitt \ref{AllgBeschreibLösung} beschrieben, wird der Quellcode zur
Darstellung der \gui mit Hilfe eines speziellen
\emph{Generators} erzeugt. Daraus folgt, dass die Integration eines neuen
Frameworks (siehe Anforderung \emph{\hyperref[AA2]{AA2}}) an die Implementierung eines
spezifischen \emph{Generators} gekoppelt ist.
In Abbildung \ref{Abb_neuerAnsatz} wird das grundlegende Konzept des
Ansatzes schematisch dargestellt. Dabei wurden exemplarisch drei
unterschiedliche \emph{Generatoren} für bestimmte Frameworks verwendet.
\myBigFigure{neuerAnsatz.png}{Grundlegendes Konzept}{neuerAnsatz}
\noindent
Der Prototyp wird aus einem \emph{GUI-Skript} den Quellcode erzeugen, welcher in
das \MCF eingebunden werden kann. Somit lässt sich prüfen, ob die \g für
existierende \guis von \pcs genügt. In Anlehnung an Abbildung
\ref{Abb_mdsd} in Kapitel \ref{mdsd} wird zur Erzeugung des Quellcodes für das
\MCF auch individueller Quellcode von Nöten sein.
In Abbildung \ref{Abb_genMCFCode} wird diese grundlegende Idee schematisch
aufgezeigt.
